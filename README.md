# Aviation Engine Health Monitoring on the NGAFID Benchmark

**Anomaly Detection (AD) and Fault Classification (FC) on a real Cessna-172 fleet**, using
classical machine learning on flight-level statistical features.

CS3315 — Introduction to Machine Learning & Big Data · Naval Postgraduate School · MSCS Q3 2026
Author: **Iván Barbosa Mejía** · Repo: <https://github.com/Barbosa12/NGAFID>

## 1. What this project does

General aviation still relies on calendar-based maintenance, which misses faults that
emerge between fixed inspection windows. Using the **National General Aviation Flight
Information Database (NGAFID)** — 11,446 Cessna-172 flights, 23 sensors @ 1 Hz, 19 fault
classes — we frame engine health monitoring as a two-stage pipeline:

1. **Anomaly Detection (AD)** — binary: *healthy* vs *anomalous*.
2. **Fault Classification (FC)** — 19-class: which fault type, given an anomalous flight.
3. **End-to-End (E2E)** — the composed 20-class diagnosis.

Five classical models (LR, Linear SVM, XGBoost, Random Forest, kNN) are evaluated over a
hierarchy of feature representations and compared against deep-learning figures reported
in the literature.

### Headline results (5-fold CV, from notebooks 03–05)

| Task | Best classical (this work) | DL reference¹ |
|------|----------------------------|---------------|
| AD   | **F1 = 0.703** (tuned Ensemble, Repr. F++) | 0.799 (ConvTokMHSA) |
| FC   | **F1 = 0.207** flat / **0.459** Oracle K=4 ceiling | 0.623 (MMK Net) |
| E2E  | **F1 = 0.131** (cascade AD→FC) | 0.623 (LMSD) |

> ¹ The deep-learning figures come from the LMSD/DDF preprint
> ([arXiv:2603.22885](https://arxiv.org/html/2603.22885v1)), which was **subsequently
> withdrawn by its authors** over acknowledged methodological flaws in the experimental
> validation and metric computation. They are reported here as *indicative* references,
> not validated upper bounds.

## 4. Get the data

The raw flight data is **not** in the repository (too large). Download it from Zenodo and
build the fixed-length tensor the notebooks expect:

```bash
# 1) Download 2days.tar.gz from the NGAFID record on Zenodo:
#    https://zenodo.org/records/6624956
# 2) Extract flight_data.pkl into data/  (flight_header.csv and stats.csv are
#    already included in the repo, but the archive also contains them):
tar -xzf 2days.tar.gz                      # → flight_data.pkl, flight_header.csv, stats.csv
mv flight_data.pkl data/

# 3) Build the fixed-length cache the notebooks load (~2 GB, a few minutes):
python scripts/build_cache.py              # → data/X_raw_L2048.npy
```

`2days.tar.gz` contains `flight_data.pkl` (variable-length raw flights),
`flight_header.csv`, and `stats.csv`. The notebooks consume a fixed-length tensor
`X_raw_L2048.npy` (each flight cubic-spline resampled to 2,048 steps); since that file is
*not* in the archive, `scripts/build_cache.py` regenerates it from `flight_data.pkl`.

The small metadata (`flight_header.csv`, `stats.csv`, `fault_mapping.json`) is already
committed in `data/` for convenience.

- Original NGAFID dataset (Zenodo): <https://zenodo.org/records/6624956>
- DDF/LMSD framework paper & source code: <https://arxiv.org/html/2603.22885v1>

## 5. How to run

First complete the data step (§4) so that `data/X_raw_L2048.npy` exists. Then launch
Jupyter from the repository root (paths are relative to it):
```bash
jupyter lab        # or: jupyter notebook
```
Run the notebooks **in order** (*Kernel → Restart & Run All* on each):

| Notebook | Needs | Produces |
|----------|-------|----------|
| `01_EDA` | `data/X_raw_L2048.npy`, `data/flight_data.pkl`, metadata | figures in `figures/` |
| `02_Dataset_Build` | same as 01 | feature representations in `datasets/*.npz` |
| `03_AD_Models` | `datasets/*.npz` | `results/ad_results.csv`, AD figures |
| `04_FC_Models` | `datasets/*.npz`, `data/fault_mapping.json` | `results/fc_results.csv`, FC figures |
| `05_E2E_Models` | `datasets/repr_Fpp_*.npz`, metadata | `results/e2e_results.csv`, E2E figures |

> To run only the modeling stage (03–05) you must first run 02 to generate `datasets/`.
> Run *Kernel → Restart & Run All* on each notebook.
